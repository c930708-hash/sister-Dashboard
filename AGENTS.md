# AI 작업 규칙

이 저장소는 ChatGPT, Codex, Claude가 이어받아 작업할 수 있으므로 아래 원칙을 지킨다.

## 기본 원칙
- GitHub `main` 최신 상태를 기준으로 작업한다.
- 기존 기능을 깨지 않는다.
- 새 기능은 현재 단일 `index.html` 구조와 화면 톤에 맞게 설계한다.
- 큰 변경은 `CHANGELOG.md`에 기록한다.
- 다음 작업이나 남은 리스크는 `TODO.md`에 최신 상태로 남긴다.
- 다른 AI 또는 사용자가 만든 변경은 임의로 되돌리지 않는다.
- 더 좋은 구조가 있으면 기존 설계도 개선하되, 사용자 데이터 저장 구조는 하위 호환을 고려한다.

## 데이터 원칙
- 예산의 계좌별 잔고와 투자 포트폴리오는 월별 이력 데이터다.
- 7월 데이터를 수정해도 6월 스냅샷이 바뀌지 않아야 한다.
- 투자 탭의 평가금액 변경은 선택된 월의 증권계좌에만 반영한다.
- 저장 전에는 Supabase 데이터가 실제 사용자 데이터임을 전제로 조심해서 검증한다.

## 배포 원칙
- 로컬 변경 후 문법 확인을 먼저 한다.
- 커밋 전 `git status`로 변경 범위를 확인한다.
- 푸시 전 `git pull --rebase origin main`으로 최신 main과 맞춘다.
- GitHub main에 푸시하면 Netlify가 자동 배포한다.
